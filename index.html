Why C++ for IoT?

Most microcontrollers — Arduino Uno, ESP32, STM32, Raspberry Pi Pico — are programmed in C or C++. C++ gives you direct hardware access: control pins, read sensors, manage memory manually. This is essential when your chip has only 2KB of RAM and must react in milliseconds.

Complete First Program — Blink + Sensor + Serial

first_program.ino — Full Sketch
/*
 * Natcom IoT Training — Module 1 Starter Project
 * Reads LM35 temperature sensor, blinks LED on hot reading,
 * prints data to Serial Monitor.
 */

#define LED_PIN    13
#define SENSOR_PIN A0
#define HOT_TEMP   28.0

// ── Sensor reading function
float readTempC() {
  int   raw     = analogRead(SENSOR_PIN);
  float voltage = raw * (5.0 / 1023.0);
  return voltage * 100.0;  // LM35: 10mV/°C
}

// ── Blink helper function
void alertBlink(int count) {
  for (int i = 0; i < count; i++) {
    digitalWrite(LED_PIN, HIGH); delay(150);
    digitalWrite(LED_PIN, LOW);  delay(150);
  }
}

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);
  Serial.println("=== Natcom Temp Monitor ===");
}

void loop() {
  float temp = readTempC();

  Serial.print("Temperature: ");
  Serial.print(temp, 1);
  Serial.println(" °C");

  if (temp > HOT_TEMP) {
    Serial.println("⚠ TOO HOT!");
    alertBlink(3);
  } else {
    digitalWrite(LED_PIN, LOW);
  }

  delay(1000);
}

The Three Fundamentals

V
Voltage (Volts)
The electrical "pressure" that pushes current through a circuit. Arduino runs on 5V, ESP32 on 3.3V.
I
Current (Amperes)
How much electricity flows. Arduino pin max = 40mA. Exceeding this destroys the chip.
R
Resistance (Ohms)
Opposition to current flow. Resistors protect LEDs and sensors from too much current.
P=VI
Power (Watts)
Electrical power consumed. A motor drawing 500mA at 5V = 2.5W of power.
📐 Ohm's Law — The IoT Builder's Formula
V = I × R  ·  I = V / R  ·  R = V / I

Example: LED needs 20mA (0.02A) and drops 2V. Arduino supplies 5V. Needed resistor: R = (5V - 2V) / 0.02A = 150 Ω. Use the nearest standard value: 220Ω (safe) or 150Ω (bright).

Essential Components

💡
LED
Light-Emitting Diode. Always needs a current-limiting resistor (220Ω for 5V Arduino). Long leg = anode (+), short leg = cathode (−).
Vf ≈ 2V · If ≈ 10–20mA
〰️
Resistor
Limits current. Read color bands: Brown=1, Black=0, Red=×100, Gold=±5% → 1kΩ. Use 220Ω for LEDs, 10kΩ for pull-up/pull-down.
Common: 100Ω 220Ω 1kΩ 10kΩ
🔋
Capacitor
Stores charge, filters noise on power lines. Place 100nF ceramic caps near microcontroller VCC pins to prevent voltage spikes.
100nF bypass · 10–100µF bulk
🎛️
Transistor / MOSFET
Amplifies or switches. Use a NPN transistor (BC547) or N-channel MOSFET (IRLZ44N) to drive motors, relays, and high-current loads from an Arduino pin.
BC547 · 2N2222 · IRF520
🔁
Relay Module
Electrically controlled switch. Lets a 3.3V/5V Arduino pin control 220V AC appliances (fans, lights). The coil and AC sides are isolated for safety.
5V coil · 10A 250VAC rating
📟
Breadboard
Prototype board — no soldering needed. Rows are connected internally. Power rails (+) and (−) run the full length. Inner rows connect across the gap.
830 ties · solderless

Connecting a Button (Pull-Down Resistor)

cpp
button_pulldown.ino
/*  Wiring:
 *  Button one leg  → 5V
 *  Button other leg→ Pin 7 AND 10kΩ resistor to GND
 *  (resistor "pulls" pin to 0 when button not pressed)
 */
const int BTN = 7;
const int LED = 13;

void setup() {
  pinMode(BTN, INPUT);          // external pull-down
  pinMode(LED, OUTPUT);
}

void loop() {
  if (digitalRead(BTN) == HIGH) {
    digitalWrite(LED, HIGH);   // button pressed
  } else {
    digitalWrite(LED, LOW);
  }
}

// Better: use Arduino's built-in pull-up (no resistor needed)
void setup2() {
  pinMode(BTN, INPUT_PULLUP);  // button→pin, button→GND
  // Logic is INVERTED: LOW = pressed, HIGH = released
}
⚠️ Voltage Safety Rules
• Never connect 5V directly to an ESP32 GPIO pin — it only tolerates 3.3V. Use a voltage divider.
• Never exceed 40mA per Arduino pin (use transistor for motors/relays).
• Always use GND from the same board — floating ground causes erratic readings.
• Disconnect USB before rewiring circuits with motors or relays.

What is IoT?

The Internet of Things is the network of physical objects embedded with sensors, software, and connectivity to collect and exchange data — without requiring human-to-human or human-to-computer interaction. The core idea: every physical object can become a data source and an actuator.

📌 The Simple Equation
Physical Object + Sensors + Microcontroller + Network = IoT Device

A temperature sensor alone is not IoT. A temperature sensor that sends its reading to the internet every 10 seconds, where a dashboard reads it and turns on a fan automatically — that is IoT.

Key IoT Protocols — The Communication Stack

MQTT
🔁 Publish / Subscribe
The #1 IoT messaging protocol. Devices publish to topics, servers subscribe to receive. Runs on TCP port 1883. Extremely lightweight — works on 8KB RAM devices.
HTTP / REST
📤 Request / Response
Familiar web protocol. Device sends HTTP POST to a server API. Simple but heavier than MQTT. Good when data is sent infrequently (every few minutes).
WebSocket
⚡ Full-Duplex
Persistent two-way connection between browser and server. Perfect for live dashboards — server pushes updates the moment new sensor data arrives.
CoAP
📦 Constrained
Constrained Application Protocol. Like HTTP but runs over UDP. Designed for devices with no TCP stack. Used in NB-IoT sensors.
AMQP
🏭 Enterprise
Advanced Message Queuing Protocol. Used in enterprise IoT (Azure Service Bus, RabbitMQ). Reliable delivery with acknowledgements.
OPC-UA
🏗️ Industrial
Standard for industrial automation and IIoT (Industry 4.0). Used in factories — connects PLCs, robots, and CNC machines to cloud systems.

Sensors — Reading the Physical World

🌡️
DHT22 — Temp + Humidity
Digital single-wire sensor. Reads −40°C to 80°C and 0–100% RH. More accurate than DHT11.
3 pins: VCC 3.3–5V · GND · DATA→any GPIO
🌊
BMP280 — Pressure + Altitude
I²C/SPI sensor measures barometric pressure and temperature. Calculate altitude from pressure difference.
I²C: SDA + SCL · Addr: 0x76 or 0x77
📏
HC-SR04 — Ultrasonic Distance
Emits 40kHz pulse, measures echo return time. Range: 2cm–400cm. Good for level detection.
TRIG→GPIO (output) · ECHO→GPIO (input)
🚶
PIR HC-SR501 — Motion
Detects infrared radiation from warm bodies. Output goes HIGH for 3–300 seconds (adjustable). Covers ~120° angle.
VCC 5V · GND · OUT→GPIO digital
💨
MQ-2 — Gas / Smoke
Detects LPG, propane, hydrogen, smoke. Analog output proportional to gas concentration. Needs 24h burn-in.
AOUT→analog pin · DOUT→digital threshold
☀️
LDR — Light Level
Resistance decreases as light increases. Use in voltage divider with 10kΩ resistor. Map 0–1023 to 0–100% brightness.
Analog voltage divider → A0 pin
🌿
Capacitive Soil Moisture
Measures soil water content using capacitance (not resistance like older resistive sensors — doesn't corrode). Essential for smart farming.
3.3–5V · Analog out 0–3.3V
📍
Neo-6M GPS Module
Outputs NMEA sentences over UART (9600 baud). Parse with TinyGPS++ library. Gets fix in ~30s outdoors. Essential for asset tracking.
TX→RX of MCU · RX→TX of MCU · 3.3V

Reading Sensors — Code Examples

DHT22 — Temperature & Humidity

cpp / ESP32
dht22_read.ino
#include <DHT.h>

#define DHT_PIN  4
#define DHT_TYPE DHT22

DHT dht(DHT_PIN, DHT_TYPE);

void setup() {
  Serial.begin(115200);
  dht.begin();
}

void loop() {
  float temp = dht.readTemperature();   // °C
  float hum  = dht.readHumidity();      // %
  float heatIndex = dht.computeHeatIndex(temp, hum, false);

  if (isnan(temp) || isnan(hum)) {
    Serial.println("ERROR: DHT22 read failed. Check wiring!");
    return;
  }

  Serial.printf("Temp: %.1f°C  Hum: %.1f%%  HeatIdx: %.1f°C\n",
               temp, hum, heatIndex);
  delay(2000);
}

HC-SR04 — Ultrasonic Distance

cpp
ultrasonic.ino
#define TRIG_PIN 5
#define ECHO_PIN 18

float getDistance() {
  // Send 10μs pulse on TRIG
  digitalWrite(TRIG_PIN, LOW);  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH); delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);

  // Measure echo duration
  long duration = pulseIn(ECHO_PIN, HIGH, 30000); // 30ms timeout

  if (duration == 0) return -1;  // timeout = out of range

  // Distance = (time × speed of sound) / 2
  // Speed of sound ≈ 0.0343 cm/μs
  return duration * 0.0343 / 2.0;
}

void loop() {
  float dist = getDistance();
  if (dist > 0) {
    Serial.printf("Distance: %.1f cm\n", dist);
    if (dist < 20) Serial.println("⚠ Object too close!");
  }
  delay(500);
}

Servo myServo;

void setup() {
  myServo.attach(SERVO_PIN);

void loop() {
  // Sweep 0° to 180° and back
  for (int angle = 0; angle <= 180; angle += 5) {
    myServo.write(angle);
    delay(30);
  }
  for (int angle = 180; angle >= 0; angle -= 5) {
    myServo.write(angle);
    delay(30);

⚠️ LoRa Duty Cycle Rules
In most regions, LoRa devices must not transmit more than 1% of the time (1% duty cycle on 868MHz EU band). This means max ~36 seconds of transmission per hour. For 20-byte packets at SF7, you can send roughly 1 packet every 6 seconds. Design your payload accordingly — send only essential data, compressed.

    global motion_last_seen

        temp  = therm.get_temperature()
        co2v  = co2.get_value()

            ac.off()

        elif PT_IoT.time() - motion_last_seen > LOCK_AFTER:

#include <WiFi.h>
#include <PubSubClient.h>
#include <DHT.h>
#include <ArduinoJson.h>

// ── Configuration ─────────────────────────────────────
#define WIFI_SSID   "YourWiFiName"
#define WIFI_PASS   "YourWiFiPassword"
#define MQTT_BROKER "broker.hivemq.com"
#define MQTT_PORT    1883
#define MQTT_TOPIC   "natcom/room1/sensors"
#define CLIENT_ID    "natcom-esp32-001"

#define DHT_PIN      4
#define PIR_PIN      14
#define LED_PIN      2    // built-in LED
#define INTERVAL_MS  10000  // send every 10 seconds

DHT dht(DHT_PIN, DHT22);
WiFiClient   wifiClient;
PubSubClient mqtt(wifiClient);

unsigned long lastSend = 0;

// ── WiFi connect ──────────────────────────────────────
void connectWiFi() {
  Serial.print("Connecting to WiFi");
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500); Serial.print(".");
  }
  Serial.printf("\n✓ WiFi connected. IP: %s\n",
               WiFi.localIP().toString().c_str());
}

// ── MQTT connect ──────────────────────────────────────
void connectMQTT() {
  while (!mqtt.connected()) {
    Serial.print("Connecting MQTT...");
    if (mqtt.connect(CLIENT_ID)) {
      Serial.println("connected!");
      mqtt.subscribe("natcom/room1/cmd");  // listen for commands
    } else {
      Serial.printf("failed rc=%d — retry in 5s\n", mqtt.state());
      delay(5000);
    }
  }
}

// ── Handle incoming commands ──────────────────────────
void onMessage(char* topic, byte* payload, unsigned int len) {
  String msg;
  for (int i = 0; i < len; i++) msg += (char)payload[i];
  Serial.printf("CMD [%s]: %s\n", topic, msg.c_str());
  if (msg == "blink") {
    for (int i = 0; i < 5; i++) {
      digitalWrite(LED_PIN, !digitalRead(LED_PIN)); delay(200);
    }
  }
}

void setup() {
  Serial.begin(115200);
  pinMode(PIR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  dht.begin();
  connectWiFi();
  mqtt.setServer(MQTT_BROKER, MQTT_PORT);
  mqtt.setCallback(onMessage);
}

void loop() {
  if (!mqtt.connected()) connectMQTT();
  mqtt.loop();

  if (millis() - lastSend >= INTERVAL_MS) {
    lastSend = millis();

    float temp   = dht.readTemperature();
    float hum    = dht.readHumidity();
    bool  motion = digitalRead(PIR_PIN);

    if (isnan(temp)) { Serial.println("DHT read error"); return; }

    // Build JSON: {"temp":24.5,"hum":62.0,"motion":false,"rssi":-68}
    StaticJsonDocument<128> doc;
    doc["temp"]   = round(temp * 10) / 10.0;
    doc["hum"]    = round(hum  * 10) / 10.0;
    doc["motion"] = motion;
    doc["rssi"]   = WiFi.RSSI();   // signal strength

    char buf[128];
    serializeJson(doc, buf);

    bool ok = mqtt.publish(MQTT_TOPIC, buf);
    Serial.printf("%s %s\n", buf, ok ? "✓" : "FAIL");

    // Flash LED to confirm send
    digitalWrite(LED_PIN, HIGH); delay(50); digitalWrite(LED_PIN, LOW);
  }
}

from flask_socketio import SocketIO
import paho.mqtt.client as mqtt_client
import json, threading, sqlite3, time

app = Flask(__name__)
sio = SocketIO(app, cors_allowed_origins="*")

# ── In-memory cache of last 100 readings ──────────────
readings = []

def on_mqtt_message(client, userdata, msg):
    try:
        data = json.loads(msg.payload)
        data['time'] = time.strftime("%H:%M:%S")
        readings.append(data)
        if len(readings) > 100: readings.pop(0)
        # Push to ALL connected browsers instantly!
        sio.emit('sensor_update', data)
    except: pass

def start_mqtt():
    c = mqtt_client.Client("natcom-server")
    c.on_message = on_mqtt_message
    c.connect("broker.hivemq.com", 1883)
    c.subscribe("natcom/#")  # subscribe to all natcom topics
    c.loop_forever()

@app.route('/')
def index():
    return render_template('dashboard.html')

@app.route('/api/history')
def history():
    return jsonify(readings[-50:])

# Start MQTT listener in background thread
threading.Thread(target=start_mqtt, daemon=True).start()

if __name__ == '__main__':
    sio.run(app, host='0.0.0.0', port=5000, debug=True)

Security Checklist for Every Deployment

  • Unique device ID and rotating API key per device — no shared secrets
  • MQTT over TLS port 8883 — never plain port 1883 in production
  • Firmware OTA via HTTPS with signature verification
  • WiFi credentials stored in NVS, never in source code
  • IoT devices on isolated VLAN — cannot reach corporate network
  • Monitor device connections — alert on unexpected new devices joining
  • All cloud API endpoints use HTTPS — never plain HTTP
  • Set MQTT "last will" message so broker knows when device disconnects unexpectedly
  • Regular firmware updates — subscribe to ESP-IDF security advisories
  • Disable Arduino OTA in production or protect it with a strong password

#include <PubSubClient.h>
#include <DHT.h>
#include <ArduinoJson.h>

// ── Change per device ──────────────────────────────
#define DEVICE_ID   "node-reception"    // node-serverroom / node-meeting
#define MQTT_TOPIC  "natcom/reception"  // natcom/serverroom / natcom/meeting

// ── Common for all nodes ───────────────────────────
#define WIFI_SSID   "NatcomIoT"
#define WIFI_PASS   "NatcomWiFiPass"
#define BROKER      "192.168.1.50"   // Raspberry Pi IP
#define BROKER_PORT 1883
#define MQTT_USER   "iotdevice"
#define MQTT_PASS   "securepass99"
#define DHT_PIN     4
#define PIR_PIN     14
#define MQ2_PIN     34
#define LED_PIN     2
#define SEND_MS     15000

DHT dht(DHT_PIN, DHT22);
WiFiClient   wc;
PubSubClient mqtt(wc);
unsigned long lastSend = 0;

void setup() {
  Serial.begin(115200);
  pinMode(PIR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  dht.begin();
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  while (WiFi.status() != WL_CONNECTED) { delay(400); }
  Serial.println("WiFi connected");
  mqtt.setServer(BROKER, BROKER_PORT);
}

void reconnect() {
  while (!mqtt.connected()) {
    if (!mqtt.connect(DEVICE_ID, MQTT_USER, MQTT_PASS))
      delay(3000);
  }
}

void loop() {
  if (!mqtt.connected()) reconnect();
  mqtt.loop();

  if (millis() - lastSend >= SEND_MS) {
    lastSend = millis();
    float temp   = dht.readTemperature();
    float hum    = dht.readHumidity();
    int   gas    = analogRead(MQ2_PIN);
    bool  motion = digitalRead(PIR_PIN);

    StaticJsonDocument<160> doc;
    doc["id"]     = DEVICE_ID;
    doc["temp"]   = isnan(temp) ? 0 : round(temp*10)/10.0;
    doc["hum"]    = isnan(hum)  ? 0 : round(hum*10)/10.0;
    doc["gas"]    = gas;
    doc["motion"] = motion;
    doc["rssi"]   = WiFi.RSSI();

    char buf[160];
    serializeJson(doc, buf);
    bool ok = mqtt.publish(MQTT_TOPIC, buf);
    digitalWrite(LED_PIN, HIGH); delay(80); digitalWrite(LED_PIN, LOW);
    Serial.println(ok ? "Sent ✓" : "FAILED");
  }
}

from flask_socketio import SocketIO
import paho.mqtt.client as mqtt_client
import json, threading, sqlite3, time, requests

app = Flask(__name__)
sio = SocketIO(app, cors_allowed_origins="*")

conn = sqlite3.connect('capstone.db', check_same_thread=False)
conn.execute("""CREATE TABLE IF NOT EXISTS readings(
  id INTEGER PRIMARY KEY, ts REAL, room TEXT,
  temp REAL, hum REAL, gas INTEGER, motion INTEGER, rssi INTEGER
)""")
conn.commit()

THRESHOLDS = {
  "node-serverroom": {"temp": 29, "gas": 2000},
  "node-reception":  {"temp": 34, "gas": 2500},
  "node-meeting":    {"temp": 32, "gas": 2500},
}
alert_cooldown = {}

TELEGRAM_TOKEN = "YOUR_BOT_TOKEN"
TELEGRAM_CHAT  = "YOUR_CHAT_ID"

def send_alert(device, msg):
    if time.time() - alert_cooldown.get(device, 0) < 300: return
    alert_cooldown[device] = time.time()
    print(f"ALERT: {msg}")
    sio.emit('alert', {'device': device, 'message': msg})
    requests.post(
        f"https://api.telegram.org/bot{TELEGRAM_TOKEN}/sendMessage",
        data={"chat_id": TELEGRAM_CHAT, "text": msg}, timeout=5
    )

def check_thresholds(data):
    dev = data.get('id', '')
    thr = THRESHOLDS.get(dev, {})
    if data.get('temp', 0) > thr.get('temp', 999):
        send_alert(dev, f"🌡 High temp {data['temp']}°C in {dev}")
    if data.get('gas', 0)  > thr.get('gas', 999):
        send_alert(dev, f"💨 Gas/smoke detected in {dev}: {data['gas']}")

def on_message(client, userdata, msg):
    try:
        data = json.loads(msg.payload)
        data['ts'] = time.strftime("%H:%M:%S")
        conn.execute(
            "INSERT INTO readings(ts,room,temp,hum,gas,motion,rssi) VALUES(?,?,?,?,?,?,?)",
            (time.time(), data.get('id'), data.get('temp'),
             data.get('hum'), data.get('gas'),
             int(data.get('motion', False)), data.get('rssi')))
        conn.commit()
        check_thresholds(data)
        sio.emit('sensor_update', data)
    except Exception as e:
        print(f"Error: {e}")

def start_mqtt():
    c = mqtt_client.Client("capstone-server")
    c.on_message = on_message
    c.username_pw_set("iotdevice", "securepass99")
    c.connect("localhost", 1883)
    c.subscribe("natcom/#")
    c.loop_forever()

Raspberry Pi Setup Guide

  5. Create auth: sudo mosquitto_passwd -c /etc/mosquitto/passwd iotdevice
  6. Edit config: sudo nano /etc/mosquitto/mosquitto.conf → add allow_anonymous false and password_file /etc/mosquitto/passwd
  7. Restart: sudo systemctl restart mosquitto && sudo systemctl enable mosquitto

  9. Clone your code from GitHub and run: python3 capstone_server.py

The most common cause is an MQTT keepalive timeout. If mqtt.loop() is not called every iteration, or reconnect() is not triggered when connected() returns false, the client silently stops publishing. Always call mqtt.loop() in every loop() iteration and always check mqtt.connected() before publishing.
